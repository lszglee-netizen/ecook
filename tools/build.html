Site builder

Regenerates per-slug HTML pages and sitemap.xml from your data files.

1. Run the build

This reads recipes.js and the other data files currently on this site, and produces a ZIP of all generated pages + a fresh sitemap.xml.

2. Apply the output

Download the ZIP, extract it to the repo root, and commit. Existing folders are overwritten with fresh content; nothing else is touched.

Quick check: the only files the build creates or overwrites are:

  • recipes/<slug>/index.html (one per non-draft recipe)
  • techniques/<slug>/index.html
  • insider/<slug>/index.html
  • reviews/<slug>/index.html
  • troubleshoot/<slug>/index.html
  • glossary/<slug>/index.html
  • notebook/<slug>/index.html
  • sitemap.xml

Notes

If you'd rather build from the command line, run python3 tools/build.py from the repo root. It does the same thing.

The GitHub Action at .github/workflows/build.yml runs this on every push, so once you set up the workflow you can also just push — Action does it for you.
